OpenXRD / Repo-First Package

Research-ready XRD evaluation without the legacy sprawl.

OpenXRD is a small public package for benchmarking language models on crystallography and X-ray diffraction questions. Install locally from the repo, acknowledge the dataset policy, unzip the benchmark, set an API key, and run a short OpenAI or OpenRouter evaluation.

Install

The recommended path is repo-first: bootstrap a local environment and install the editable package in one step.

git clone https://github.com/niaz60/OpenXRD.git cd OpenXRD ./scripts/install.sh source .venv/bin/activate

Unlock The Dataset

The benchmark stays zipped in the public repo. Extraction requires an explicit acknowledgment of the evaluation-only policy.

./scripts/unzip_dataset.sh --acknowledge openxrd-check

Run A Short Example

Set either an OpenAI key or an OpenRouter key, choose a model explicitly, and run a small evaluation sample.

openxrd-example --provider openai --model your-openai-model --limit 3

Provider Setup

OpenRouter can be used for Claude-family models through the OpenAI-compatible interface.

OpenAI

Generate a key at platform.openai.com/api-keys.

export OPENAI_API_KEY="your-openai-key" openxrd-example --provider openai --model your-openai-model --limit 3

OpenRouter

Generate a key at openrouter.ai/keys.

export OPENROUTER_API_KEY="your-openrouter-key" openxrd-example --provider openrouter --model anthropic/claude-3.5-sonnet --limit 3

Dataset Policy

The archive is intentionally frictional, not access-controlled.

Why It Is Zipped

The public dataset is distributed as datasets/openxrd_dataset.zip to reduce casual scraping and accidental ingestion by online aggregation or crawling tools.

  • It adds friction, not strong protection.
  • It keeps the public repo cleaner for researchers.
  • It avoids advertising raw JSON as a default crawl target.

Allowed Use

  • Evaluation and benchmarking only.
  • No training, fine-tuning, distillation, or alignment use.
  • Citation of the accepted paper is required when using the dataset.
